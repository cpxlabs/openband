# Sample Browser

## Overview
OpenBand lets users browse built-in sample packs and add them to tracks. The `SampleBrowser` component (`src/components/SampleBrowser.tsx`) renders a searchable, category-filtered grid of sample entries and supports click-to-preview playback. Debounced preview audio is produced by the `PreviewEngine` in `src/lib/previewEngine.ts` (`previewAsset`, `generateThumbnail`), which currently has no spec. Sample thumbnails are generated by extracting a short slice of the full audio and re-rendering it via an `OfflineAudioContext`.

## Implementation Notes
`SampleBrowser.tsx` holds a static `SAMPLES` array of `SampleEntry` objects (`{ id, name, category, color, duration }`) and a `SAMPLE_CATEGORIES` list (`src/components/SampleBrowser.tsx:21`). Preview playback synthesizes a cached WAV per sample id via `OfflineAudioContext` + `audioBufferToWavBlob` (`src/components/SampleBrowser.tsx:558`), since no remote sample binaries ship. The `PreviewEngine` (`src/lib/previewEngine.ts`) manages a shared debounced `AudioContext` source, exposes `PreviewAsset` (`{ id, name, fullUrl, thumbnailUrl, duration, sampleRate, channels, category }`), `PreviewState`, and `previewAsset`/`stopPreview`/`setPreviewVolume`/`onPreviewStateChange`/`generateThumbnail`. `SampleBrowser` itself uses `expo-audio` directly and does not yet route through `PreviewEngine` for its previews.

## Requirements

### Requirement: Browse Sample Packs
The system MUST provide a sample browser that lists available sample packs with search and category filtering, rendering each entry with its name, category icon, and duration.

#### Scenario: Filter by category
- **Given** the full `SAMPLES` list in `SampleBrowser.tsx:33`
- **When** the user selects the "808" category
- **Then** only `SampleEntry` items with `category === "xx808"` are shown
- **And** other categories are hidden

#### Scenario: Search by name
- **Given** the sample grid is visible
- **When** the user types "Kick" into the search field
- **Then** only samples whose `name` includes "kick" (case-insensitive) remain

### Requirement: Click-to-Preview with Debounce
The system MUST allow users to preview a sample by selecting it, and the preview audio MUST be debounced so rapid selections do not stack playback.

#### Scenario: Debounced preview restart
- **Given** the `PreviewEngine` with default `debounceMs = 150`
- **When** `previewAsset(assetA)` is called, then `previewAsset(assetB)` within 150ms
- **Then** only one playback starts after the timer elapses
- **And** the first pending playback is cancelled via `clearTimeout`

#### Scenario: Stop current before next
- **Given** a preview is playing
- **When** a new `previewAsset` fires
- **Then** `stopCurrentPlayback` disconnects the active `AudioBufferSourceNode` before the new source starts

### Requirement: Add Sample to Track/Region
The system MUST let a user add a browsed sample to a track as a new region/sample slot via the `onAddSample` callback.

#### Scenario: Add sample to track
- **Given** a `SampleCard` rendered from a `SampleEntry`
- **When** the user taps the "+ TRACK" button
- **Then** `onAddSample(sample)` is invoked with that `SampleEntry`
- **And** the parent creates a `TrackRegion` (or sampler slot) from the sample metadata

### Requirement: Thumbnail Generation
The system MUST be able to generate a short preview/thumbnail audio blob from a full sample URL via `generateThumbnail(fullAudioUrl, format, durationSec)`.

#### Scenario: Generate 2s thumbnail
- **Given** a full sample URL decodable by the shared `AudioContext`
- **When** `generateThumbnail(url, "mp3", 2)` is called
- **Then** the first 2 seconds are sliced, re-rendered through an `OfflineAudioContext`, and returned as a WAV `Blob`
- **And** the result is `null` when `Platform.OS !== "web"` or the fetch/decode fails

### Requirement: Preview State Subscription
The system MUST expose the current preview state (`currentAsset`, `isPlaying`, `volume`, `loading`, `bufferTime`) and notify subscribers on change via `onPreviewStateChange`.

#### Scenario: State callback fires
- **Given** a subscriber registered through `onPreviewStateChange`
- **When** `previewAsset` begins loading and then starts playback
- **Then** the callback receives `PreviewState` with `isPlaying: true` and `loading: false`
- **And** `getPreviewState()` returns the same snapshot

## Test Requirements (Vitest)
- [ ] Category filter returns only matching `SampleEntry` items
- [ ] Name search matches case-insensitively and excludes non-matching
- [ ] `previewAsset` debounce coalesces rapid calls (only one playback)
- [ ] `stopPreview` clears the debounce timer and resets `PreviewState`
- [ ] `setPreviewVolume` clamps to `[0, 1]` and updates `gainNode.gain`
- [ ] `generateThumbnail` returns a non-null WAV `Blob` for a valid URL on web
- [ ] `generateThumbnail` returns `null` on non-web platforms
- [ ] `onPreviewStateChange` callback fires on play/stop transitions
